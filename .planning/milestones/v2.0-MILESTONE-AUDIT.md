## Cross-Phase Integration Findings

### Functional Breaks (Priority 1)

**1. Builder Tab Broken for All Users**
- **Root cause:** `/api/v1/me` returns `roles: ["admin"]` (array) but `workflows.html:568` checks `CURRENT_USER.role` (singular scalar) — always `undefined`
- **Also:** `CURRENT_USER.is_superadmin` field missing from `/me` response
- **Impact:** No user can access the Builder tab. Superadmin sign-off capability broken in onboarding.
- **Affects:** WKFL-04, ONBD-04

**2. New Hire Onboarding Flow Incomplete**
- **Root cause:** `POST /api/v1/onboarding/assignTemplate` and `unassignTemplate` endpoints exist in backend but have ZERO UI callers in `onboarding.html`
- **Impact:** After a hire accepts their invite, their "My Trainings" tab is empty. No UI path exists for managers to assign training templates.
- **Affects:** ONBD-02, USER-01

**3. Delete Template 404s**
- **Root cause:** `onboarding.html:1074` calls `PUT .../updateTemplate/{id}/delete` but chi has no matching route (only `PUT /updateTemplate/{id}`)
- **Impact:** Builder "Delete Template" button silently fails
- **Affects:** ONBD-04

**4. Corrective Action Photos Not Persisted**
- **Root cause:** `handleFailPhotoCaptureClick` stores `blob:` URLs instead of routing through presign → Spaces pipeline
- **Impact:** Fail card evidence photos lost on page reload
- **Affects:** PHOT-02 (partial)
