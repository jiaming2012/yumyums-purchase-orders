**Functions to implement:**

1. `func GenerateToken() (raw string, hash string, err error)` — 32 bytes from `crypto/rand`, hex-encode for raw, SHA-256 hash for storage. Exactly as in research Pattern 3.

2. `func HashToken(raw string) string` — SHA-256 hash of raw token string, returns hex-encoded hash. Used by middleware to look up sessions.

3. `func VerifyPassword(hash, password string) error` — wraps `bcrypt.CompareHashAndPassword([]byte(hash), []byte(password))`.

4. `func HashPassword(password string) (string, error)` — wraps `bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)`, returns string.

5. `func CreateSession(ctx context.Context, pool *pgxpool.Pool, userID string) (rawToken string, err error)`:
   - Call `GenerateToken()` to get raw + hash
   - INSERT into sessions: `INSERT INTO sessions (token_hash, user_id) VALUES ($1, $2)` — no expires_at (NULL = indefinite per D-03)
   - Return raw token (to be set in cookie)

6. `func DeleteSessionByHash(ctx context.Context, pool *pgxpool.Pool, tokenHash string) error`:
   - `DELETE FROM sessions WHERE token_hash = $1`

7. `func LookupSession(ctx context.Context, pool *pgxpool.Pool, tokenHash string, superadmins map[string]config.SuperadminEntry) (*User, error)`:
   - Query: `SELECT u.id, u.email, u.display_name, u.role, u.status FROM sessions s JOIN users u ON s.user_id = u.id WHERE s.token_hash = $1 AND (s.expires_at IS NULL OR s.expires_at > now())`
   - If no row found, return nil, nil (no error — just not found)
   - Scan into User struct
   - Check if user email is in superadmins map — if yes, set `IsSuperadmin = true` and override `Role` to `"superadmin"` in the returned struct
   - Return user

8. `func AuthenticateUser(ctx context.Context, pool *pgxpool.Pool, email, password string, superadmins map[string]config.SuperadminEntry) (*User, error)`:
   - Query: `SELECT id, email, display_name, password_hash, role, status FROM users WHERE email = $1`
   - If no row, return nil, nil
   - If `password_hash` is NULL (invited but not accepted), return nil, nil
   - Call `VerifyPassword(passwordHash, password)` — if error, return nil, nil
   - Build User struct. Check superadmins map for email — if found, set IsSuperadmin=true and Role="superadmin"
   - Return user

9. `func UpsertSuperadmins(ctx context.Context, pool *pgxpool.Pool, superadmins map[string]config.SuperadminEntry) error`:
   - For each superadmin entry, run:
     ```sql
     INSERT INTO users (email, display_name, role, status)
     VALUES ($1, $2, 'admin', 'invited')
     ON CONFLICT (email) DO UPDATE SET display_name = EXCLUDED.display_name
     ```
   - This ensures superadmin exists in users table (so they have a row for password_hash storage when they accept invite). Per Open Question 1 resolution: superadmin rows live in users table; superadmin privilege is an in-memory overlay.
   - Log each upserted superadmin email.

**Create `backend/internal/auth/middleware.go`:**

Package: `auth`

```go
package auth

import (
    "context"
    "net/http"
    "github.com/jackc/pgx/v5/pgxpool"
    "github.com/yumyums/hq/internal/config"
)

type contextKey string
const CtxKeyUser contextKey = "user"

// UserFromContext extracts the authenticated user from request context.
// Returns nil if no user is set (should not happen behind middleware).
func UserFromContext(ctx context.Context) *User {
    u, _ := ctx.Value(CtxKeyUser).(*User)
    return u
}

// Middleware validates the hq_session cookie, looks up the session in DB,
// and attaches the User to request context. Returns 401 if invalid.
func Middleware(pool *pgxpool.Pool, superadmins map[string]config.SuperadminEntry) func(http.Handler) http.Handler {
    return func(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            cookie, err := r.Cookie("hq_session")
            if err != nil {
                http.Error(w, `{"error":"unauthorized"}`, http.StatusUnauthorized)
                return
            }
            tokenHash := HashToken(cookie.Value)
            user, err := LookupSession(r.Context(), pool, tokenHash, superadmins)
            if err != nil || user == nil {
                http.Error(w, `{"error":"unauthorized"}`, http.StatusUnauthorized)
                return
            }
            ctx := context.WithValue(r.Context(), CtxKeyUser, user)
            next.ServeHTTP(w, r.WithContext(ctx))
        })
    }
}
```

Package: `auth`

Imports: `encoding/json`, `net/http`, `log`, `github.com/jackc/pgx/v5/pgxpool`, `github.com/yumyums/hq/internal/config`

**LoginHandler** — `func LoginHandler(pool *pgxpool.Pool, superadmins map[string]config.SuperadminEntry) http.HandlerFunc`:
- Parse JSON body: `{ "email": string, "password": string }`
- If body parsing fails or email/password empty, return 400 `{"error":"validation_error"}`
- Call `AuthenticateUser(ctx, pool, email, password, superadmins)`
- If result is nil, return 401 `{"error":"invalid_credentials"}` (per D-05: simple error)
- Call `CreateSession(ctx, pool, user.ID)` to get raw token
- Set cookie: `http.SetCookie(w, &http.Cookie{Name: "hq_session", Value: rawToken, Path: "/", HttpOnly: true, Secure: true, SameSite: http.SameSiteStrictMode})`
  - No MaxAge or Expires — per D-03: indefinite sessions
- Return 200 JSON: `{"user": { "id": user.ID, "email": user.Email, "display_name": user.DisplayName, "role": user.Role }}`
- NOTE: Do NOT return the token in the response body. The token is in the cookie only. This differs from docs/user-management-api.md which specified a `token` field — that design assumed localStorage, but we use httpOnly cookies per architecture decision.

**LogoutHandler** — `func LogoutHandler(pool *pgxpool.Pool) http.HandlerFunc`:
- Read `hq_session` cookie
- If no cookie, return 204 (idempotent — already logged out)
- Hash the token, call `DeleteSessionByHash(ctx, pool, tokenHash)`
- Clear cookie: `http.SetCookie(w, &http.Cookie{Name: "hq_session", Value: "", Path: "/", HttpOnly: true, Secure: true, SameSite: http.SameSiteStrictMode, MaxAge: -1})`
- Return 204 No Content

**MeHandler** — `func MeHandler() http.HandlerFunc`:

**MeAppsHandler** — `func MeAppsHandler(pool *pgxpool.Pool) http.HandlerFunc`:
- Get user from context
- If user is superadmin: query all enabled apps: `SELECT slug, name, icon FROM hq_apps WHERE enabled = true ORDER BY slug`
- Otherwise: query apps where user has access via role grant OR individual grant:
  ```sql
  SELECT DISTINCT a.slug, a.name, a.icon FROM hq_apps a
  LEFT JOIN app_permissions p ON p.app_id = a.id
  WHERE a.enabled = true AND (p.role = $1 OR p.user_id = $2)
  ORDER BY a.slug
  ```
- Return 200 JSON array of `{"slug", "name", "icon"}` objects
