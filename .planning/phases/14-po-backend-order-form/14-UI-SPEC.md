## Component Inventory

These components must be built for Phase 14. Each references an existing pattern to reuse or adapt.

### 1. 4-Tab Bar

```
[ Order ] [ Shopping ] [ PO ] [ History ]
```

- Renders 4 tabs: Order, Shopping, PO, History
- Only **Order** tab is fully interactive; Shopping / PO / History show empty or stub states
- Active tab: `--info-bg` background, `--info-tx` text
- Pattern: extend existing `.tabs` from `purchasing.html` with a 4th tab

### 2. Reorder Suggestion Banner

```
┌─────────────────────────────────────────┐
│  3 items need restock              [+]  │
└─────────────────────────────────────────┘
```

- Background: `--warn-bg`, text: `--warn-tx`
- Sits at the top of the Order tab, above the item list
- Hidden when no items are below threshold
- [+] button opens the suggestion checklist (see component 3)
- Pattern: adapt `.reorder-section` from `inventory.html`

### 3. Reorder Suggestion Checklist (inline below banner)

```
┌─────────────────────────────────────────┐
│  3 items need restock         [Dismiss] │
├─────────────────────────────────────────┤
│  ☐  Salmon fillet    Suggested: 5 lb   │
│  ☐  Shrimp 16/20     Suggested: 8 lb   │
│  ☐  Lemons           Suggested: 2 case │
├─────────────────────────────────────────┤
│            [Add Selected]               │
└─────────────────────────────────────────┘
```

- Toggles open/closed via the [+] button on the banner
- Each row: checkbox + item name + suggested quantity (pre-filled from threshold)
- "Add Selected" button: accent style — adds all checked items to PO at suggested qty
- "Dismiss" collapses the checklist without adding items

### 4. PO Item Card (photo card layout)

```
┌─────────────────────────────────────────┐
│ [img] Salmon fillet          [ − ] 5 [ + ] │
│       lb · Aisle 4                      │
│       Added by: JM                      │
└─────────────────────────────────────────┘
```

- Height: ~80px tall
- Left: 48×48px thumbnail photo (rounded, `object-fit:cover`); fallback: grey placeholder with item initial
- Middle: item name (14px/600), unit + store location note (12px/400/muted), "Added by: [initials]" (11px/400/muted)
- Right: stepper — `−` (32×32px circle button), quantity (14px/600, min-width 24px, center-aligned), `+` (32×32px circle button)
- Quantity 0: dim the card slightly (`opacity: 0.5`) — item stays in list but won't be submitted
- Pattern: extend `.row` + `.stp` from `purchasing.html` with photo column added on left

### 5. Category Sticky Header

```
PROTEINS
```

- `position: sticky; top: 0`
- Background: `--bg` so it covers scrolling content
- 11px, weight 600, uppercase, letter-spacing 0.5px, muted color
- Pattern: `.cat` class from `purchasing.html`

### 6. Item Picker Modal (fullscreen)

```
┌─────────────────────────────────────────┐
│ [ Search items...          ] [Back to order]   │
├─────────────────────────────────────────┤
│  Salmon fillet   lb                     │
│  ● On PO: 5 lb                          │
├─────────────────────────────────────────┤
│  Shrimp 16/20    lb       [+ Add]       │
├─────────────────────────────────────────┤
│  Lemons          case     [+ Add]       │
└─────────────────────────────────────────┘
```

- Full-screen takeover: `position:fixed; inset:0; z-index:9999; background:var(--bg)`
- Header: search input (flex:1) + "Back to order" text button (accent color)
- Search: debounce 300ms, filters item list by name (case-insensitive)
- Already-on-PO items: show "On PO: [qty]" badge in accent style (`--info-bg`/`--info-tx`), no [+ Add] button
- Not-on-PO items: show [+ Add] button — tapping adds item at qty 1 and closes modal
- Pattern: `.item-modal` from `inventory.html`

### 7. Duplicate Item Popup (inline or bottom sheet)

```
┌─────────────────────────────────────────┐
│  Salmon fillet already on PO            │
│  Qty: 5 · Added by: JM                 │
│                                         │
│  [Go to item]              [Dismiss]    │
└─────────────────────────────────────────┘
```

- Triggered when user taps an already-on-PO item in the picker
- Inline popup or bottom sheet (not a blocking modal) — use a fixed-bottom card with backdrop
- "Go to item" scrolls to that card in the Order list and closes modal
- "Dismiss" closes the popup and returns to the picker

### 8. "+ Add Item" Trigger Button

```
┌─────────────────────────────────────────┐
│           + Add item                    │
└─────────────────────────────────────────┘
```

- Full-width button at the bottom of the Order list (inside the scrollable area)
- Accent style: `--info-bg` background, `--info-tx` text
- Tapping opens the fullscreen Item Picker Modal
- Pattern: `.btn-primary` / `.add-line-btn` from `inventory.html`

### 9. Order Tab Header Card

```
┌─────────────────────────────────────────┐
│  Week of Apr 20                         │
│  3 contributors · 7 items   [Cutoff Sun 6pm] │
└─────────────────────────────────────────┘
```

- Shows week label, contributor count, item count
- Cutoff pill: `--warn-bg`/`--warn-tx` (amber) — "Cutoff [day] [time]"
- Pattern: `.hd` + `.sub` + `.pill` from `purchasing.html`

### 10. Empty State (Order tab, no items yet)

```
┌─────────────────────────────────────────┐
│                                         │
│      🛒                                 │
│      Nothing on the order yet           │
│      Add items from inventory or        │
│      tap the restock suggestions above  │
│                                         │
└─────────────────────────────────────────┘
```

- Centered column layout inside the card content area
- Icon: 🛒 at 32px
- Heading: 15px/600 — "Nothing on the order yet"
- Body: 14px/400/muted — instruction text
- Only shown when PO has zero line items

### 11. Stub States for Shopping / PO / History tabs

- **Shopping:** Single card with muted text — "Shopping list will appear here after the PO is approved"
- **PO:** Single card with muted text — "Locked PO will appear here after the weekly cutoff"
- **History:** Single card with muted text — "Past shopping runs will appear here"
- Pattern: simple `.card` with centered muted text; no interactive elements in Phase 14

## Interaction Contract

### Stepper behavior
- Minimum quantity: 0 (button does not decrement below 0)
- Maximum quantity: none (no upper cap)
- Tap +/−: immediate optimistic UI update → `PATCH /api/v1/purchasing/po-line-items/{id}` with debounce 600ms
- Quantity 0: card dims (`opacity: 0.5`) — item remains in list

### Reorder suggestion banner
- Hidden if 0 items are below threshold
- Tapping [+] expands the checklist inline below the banner (not a modal)
- "Add Selected": sends selected items to API → appends cards to item list → collapses checklist
- "Dismiss": collapses checklist with no changes

### Item picker search
- Debounce: 300ms after last keystroke
- Filter: case-insensitive substring match on item name
- Already-on-PO items: shown at top of list with "On PO" badge, tapping shows duplicate popup
- Not-on-PO items: shown with [+ Add] button, tapping adds at qty 1 and closes modal

### Duplicate detection
- Checked client-side against current PO state before calling API
- Popup shown immediately (no API roundtrip needed for detection)
- "Go to item" scrolls the Order list to that card (`scrollIntoView({ behavior: 'smooth' })`) then closes modal

### Loading and error states
- Initial load: show "Loading order..." centered muted text while awaiting API response
- API error: show inline error card with retry message — no blocking alert()
- Optimistic quantity updates: update DOM immediately, roll back on API error

### Tab switching
- `show(n)` function toggles `style.display` on `s1`–`s4` sections
- Shopping / PO / History tabs: render stub state immediately (no API call needed)

## Accessibility Contract

- All interactive elements: minimum touch target 44×44px (stepper buttons are 32×32px circle — wrap in a larger tap zone or accept existing pattern, consistent with rest of app)
- No `outline: none` without a visible focus replacement
- Thumbnails: `alt="[item name]"` always set
- Item picker search input: `autofocus` when modal opens
- Quantity span: `aria-label="Quantity: [N]"` on each stepper span
