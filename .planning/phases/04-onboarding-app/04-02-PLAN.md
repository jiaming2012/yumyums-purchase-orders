**New functions:**

`renderManager()` — branches on `mgrState.view`:
- If 'list': render sub-tabs ("Active" / "Completed") and call `renderMgrList()`
- If 'runner': render back link + read-only checklist runner

`renderMgrList()` — branches on `mgrState.subView`:
- 'active': filter `MOCK_HIRES` to those with at least one template not fully complete. For each hire, render a `.card` with:
  - Hire name (14px weight 500) + role pill (`.pill`, text = hire.role with underscores replaced by spaces, capitalized)
  - Start date (12px var(--mut), format: "Started Apr 1, 2026")
  - For each assigned template: progress bar (4px) + "X% — X/Y tasks" text (per D-02)
  - Card is tappable: `data-action="open-hire"`, `data-hire-id="..."`, `data-tpl-idx="0"`
  - Below the card: "+ Assign Training" link (`data-action="show-assign"`, `data-hire-id="..."`)
- 'completed': filter to hires where ALL assigned templates are fully complete (all sections signed_off and FAQ viewed). Show same card layout but no assign button. If none, show empty state: "No completed onboardings yet." / "Finished onboardings will show here after all sections are signed off."

Active empty state: "No active onboarding." / "New hires will appear here when added to the system."

`renderMgrRunner(hireId, tplIdx)` — renders the same section/item layout as `renderRunner()` from Plan 01 but with `readOnly: true`. This means:
- Checkboxes render as static display (checked or unchecked appearance) with NO `data-action` attribute — manager cannot toggle them (Pitfall 4)
- Video "Watch Video" links still work (read-only doesn't block link viewing)
- FAQ sections expand/collapse normally (read-only)
- Sections with state `pending_signoff` show TWO buttons at the section bottom:
  - "Approve Section" button (`.btn-approve`, `data-action="approve-signoff"`, `data-hire-id`, `data-section-id`)
  - "Send Back" button (`.btn-danger`, `data-action="send-back"`, `data-hire-id`, `data-section-id`)
- Sections with state `signed_off` show "Signed Off ✓" in green text
- Back link text: "&#8249; All Hires"

**Refactor renderRunner():** The existing `renderRunner()` from Plan 01 renders the crew view. To avoid duplicating the entire runner, refactor it to accept a `readOnly` parameter:
- `function renderRunnerContent(hireId, tplIdx, readOnly)` — shared rendering logic
- `renderRunner()` calls `renderRunnerContent(obState.activeHireId, obState.activeTemplateIdx, false)` and puts result into `#my-body`
- `renderMgrRunner()` calls `renderRunnerContent(mgrState.activeHireId, mgrState.activeTemplateIdx, true)` and puts result into `#mgr-body`
- When `readOnly === true`: checkbox divs have no `data-action`, no cursor:pointer; "Request Sign-Off" button is hidden; approve/send-back buttons appear for pending_signoff sections

**Assign Training picker:**
When manager taps "+ Assign Training", render an `.assign-picker` div below the button that lists `MOCK_OB_TEMPLATES` NOT already in that hire's `assignedTemplates`. Each row: template name, `data-action="assign-training"`, `data-hire-id`, `data-tpl-id`. If no unassigned templates available, show "No additional trainings available." in muted text.

When a template row is tapped: push the template ID into `hire.assignedTemplates`, call `initSectionStates(hire)` for the new template, re-render.

**Event delegation on `#mgr-body` (add ONE click listener, outside render functions):**
- `switch-sub`: toggle `mgrState.subView` between 'active'/'completed', re-render
- `open-hire`: set `mgrState = { view: 'runner', subView: mgrState.subView, activeHireId, activeTemplateIdx }`, `renderManager()`
