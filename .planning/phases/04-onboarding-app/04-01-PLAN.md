**Mutable state dicts:**
```js
const SECTION_STATES = {};  // SECTION_STATES[hireId][sectionId] = 'locked'|'active'|'pending_signoff'|'signed_off'
const OB_CHECKS = {};       // OB_CHECKS[itemOrPartId] = { checked: true, checkedBy: 'Name', checkedAt: Date }
const FAQ_VIEWED = {};       // FAQ_VIEWED[hireId] = true
const EXPANDED_SECTIONS = {}; // EXPANDED_SECTIONS[sectionId] = true (UI expand/collapse state)
const EXPANDED_FAQS = {};    // EXPANDED_FAQS[faqItemId] = true (individual Q&A expand state)
```

**Navigation state:**
```js
let obState = { view: 'list', activeHireId: null, activeTemplateIdx: 0 };
```

**Helper functions:**

`isManager()` — returns `['admin','manager','superadmin'].includes(MOCK_CURRENT_USER.role)`

`getMyHire()` — returns `MOCK_HIRES.find(h => h.userId === MOCK_CURRENT_USER.id)` (the logged-in user's hire record, if any)

`initSectionStates(hire)` — for each assigned template, for each section, set first section to 'active', rest to 'locked'. Skip if already initialized for this hire.

`isSectionComplete(hireId, section)` — returns true if:
  - For isFaq sections: `FAQ_VIEWED[hireId] === true`
  - For regular sections: all items are checked (checkbox items have OB_CHECKS[itemId], video_series items have all parts checked)

`canUnlockNext(hireId, sections, currentIndex)` — check if current section is complete AND (if requiresSignOff) is in 'signed_off' state. If both true, unlock next section by setting its state to 'active'.

`getProgress(hireId, template)` — counts total checkable items (checkboxes + video parts across all non-FAQ sections) and checked items. Returns `{ checked, total, pct }`.

`tryAdvanceSections(hireId, tplId)` — after any state mutation, iterate sections in order and unlock the next locked section if the previous is fully complete (and signed off if required). This is the single place where section transitions happen.

**Render functions:**

`renderMyTrainings()` — branches on `obState.view`:
  - If 'list': calls `renderMyList()`
  - If 'runner': calls `renderRunner()`

`renderMyList()` — finds the current user's hire record via `getMyHire()`. If no hire, show empty state: "No trainings yet." / "Your onboarding checklist will appear here once assigned by your manager." If hire exists, for each assigned template, render a `.card` with: template name (14px weight 500), progress bar (4px, #27ae60 fill), "X of Y tasks complete" (12px var(--mut)), `data-action="open-checklist"` + `data-tpl-idx="N"`.

`renderRunner()` — shows the checklist runner for `obState.activeHireId` at template index `obState.activeTemplateIdx`. At the top: a back link `<span class="back" data-action="back-to-list">&#8249; My Trainings</span>`, then the template name as a heading. Then for each section:
  - Section header: `.sec-header` with section title, progress count "[X/Y]", and state badge (locked/pending/signed_off). If locked, add `.locked` class (no expand affordance).
  - If section is active and expanded (`EXPANDED_SECTIONS[secId]`), render items:
    - `type: 'checkbox'`: `.item-row` with `.ob-check` box (checked class if OB_CHECKS[itemId] exists), label, attribution if checked
    - `type: 'video_series'`: render the series label, then each part as an `.item-row` with checkbox, part title, part desc (12px var(--mut)), and "Watch Video" link (`<a class="video-link" data-action="watch-video" data-url="...">Watch Video</a>`)
    - `isFaq: true` section: render FAQ section header with "View Required" pill (warn) or "Viewed ✓" pill (ok). On expand, render each Q&A pair as `.faq-q` (question with expand chevron) and `.faq-a` (answer, shown if EXPANDED_FAQS[faqId])
  - After all items in a section: if `requiresSignOff === true` AND all items checked AND state is 'active', show "Request Sign-Off" button (`.btn-primary`, `data-action="request-signoff"`, `data-section-id="..."`, `data-hire-id="..."`)
  - If section state is 'pending_signoff', show "Pending Sign-Off" status text
  - If section state is 'signed_off', show "Signed Off ✓" text in green

**Event delegation — TWO listeners (one per container, attached ONCE outside render):**

On `#my-body` click:
- `open-checklist`: set `obState = { view: 'runner', activeHireId: getMyHire().id, activeTemplateIdx: +btn.dataset.tplIdx }`, call `initSectionStates(hire)`, `renderMyTrainings()`
- `back-to-list`: set `obState = { view: 'list', ... }`, `renderMyTrainings()`
- `toggle-section`: toggle `EXPANDED_SECTIONS[secId]`, re-render. For FAQ sections, on first expand set `FAQ_VIEWED[hireId] = true`.
- `toggle-item`: toggle `OB_CHECKS[itemId]` (add or delete), call `tryAdvanceSections(hireId, tplId)`, re-render
- `toggle-video-part`: same as toggle-item but for video part IDs
- `watch-video`: `window.open(btn.dataset.url, '_blank')` — do NOT toggle the checkbox; that is a separate action
- `request-signoff`: set `SECTION_STATES[hireId][secId] = 'pending_signoff'`, call `tryAdvanceSections()`, re-render
- `toggle-faq`: toggle `EXPANDED_FAQS[faqId]`, re-render
