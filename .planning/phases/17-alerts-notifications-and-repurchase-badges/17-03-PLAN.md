<!-- From inventory/handler.go GetStockHandler SQL -->
```sql
SELECT sub.description, sub.group_name,
    COALESCE(sco.quantity, sub.total_quantity) AS total_quantity,
    sub.total_spend, sub.avg_price, sub.last_purchase_date,
    sub.low_threshold, sub.high_threshold
FROM (
    SELECT COALESCE(pi.description, pli.description) AS description, ...
    FROM purchase_line_items pli
    JOIN purchase_events pe ON pe.id = pli.purchase_event_id
    LEFT JOIN purchase_items pi ON pi.id = pli.purchase_item_id
    LEFT JOIN item_groups ig ON ig.id = pi.group_id
    GROUP BY ...
) sub
LEFT JOIN stock_count_overrides sco ON sco.item_description = sub.description
ORDER BY sub.total_spend DESC
```

    **handler.go — GetStockHandler update (per D-14, D-15):**
    - Modify the outer SELECT to include: `COALESCE(re_agg.repurchased_qty, 0) AS repurchased_qty`
    - Add a LEFT JOIN to the outer query (after the stock_count_overrides join):
      ```sql
      LEFT JOIN (
          SELECT sli.purchase_item_id, SUM(re.quantity) AS repurchased_qty
          FROM repurchase_events re
          JOIN shopping_list_items sli ON sli.shopping_list_id = re.shopping_list_id AND sli.purchase_item_id = re.purchase_item_id
          WHERE re.repurchased_at > COALESCE(
              (SELECT last_reset_at FROM badge_reset_config LIMIT 1),
              '1970-01-01'::timestamptz
          )
          GROUP BY sli.purchase_item_id
      ) re_agg ON re_agg.purchase_item_id = sub.purchase_item_id
      ```
    - IMPORTANT: The inner subquery needs to expose `purchase_item_id` for the join. Add `pi.id AS purchase_item_id` to the inner SELECT and include it in GROUP BY. The outer query can then join re_agg on sub.purchase_item_id.
    - Update rows.Scan to include `&s.RepurchasedQty`
    - Per Pitfall 3: the COALESCE fallback to '1970-01-01' ensures that if badge_reset_config has no rows, ALL repurchase events are summed (which is fine for initial state before admin configures a reset).
