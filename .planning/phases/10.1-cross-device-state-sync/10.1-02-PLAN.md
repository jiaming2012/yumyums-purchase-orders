**hub.go** — Channel-based hub (no mutex), single goroutine owns client map per D-06, D-09:

```go
package sync

type Client struct {
    UserID string
    Conn   *websocket.Conn
    Send   chan []byte
}

type BroadcastMsg struct {
    UserIDs []string
    Data    []byte
}

type Hub struct {
    clients    map[string][]*Client  // userID -> connections
    register   chan *Client
    unregister chan *Client
    broadcast  chan BroadcastMsg
}

func NewHub() *Hub { ... }

func (h *Hub) Run() {
    // select loop: register adds to map, unregister removes, broadcast fans out
    // On broadcast: iterate msg.UserIDs, send to each client's Send channel
    // If Send channel full (default case on select), close it and mark for removal
}
```

Each Client has a write pump goroutine that reads from `Send` channel and calls `conn.Write()`.

**listener.go** — Uses pgxlisten per research Pattern 2:

```go
package sync

func StartListener(ctx context.Context, connStr string, hub *Hub, pool *pgxpool.Pool) {
    listener := &pgxlisten.Listener{
        Connect: func(ctx context.Context) (*pgx.Conn, error) {
            return pgx.Connect(ctx, connStr)
        },
        ReconnectDelay: 5 * time.Second,
    }
    listener.Handle("ops_channel", pgxlisten.HandlerFunc(func(ctx context.Context, notification *pgconn.Notification, conn *pgx.Conn) error {
        // Parse notification payload JSON: { op_id, entity_id, entity_type, user_id }
        // Call GetOpByID(ctx, pool, opID) to fetch full op row
        // Call ResolveEntityAccess(ctx, pool, entityID, entityType) to get target user IDs
        // Marshal op as JSON, send BroadcastMsg{userIDs, data} to hub.broadcast channel
        return nil
    }))
    go listener.Listen(ctx)
}
```

Key: Use a DEDICATED pgx.Conn for LISTEN (not pgxpool). pgxlisten manages reconnection automatically.

**handler.go** — Two handlers per D-06, D-07, D-08:

**WsHandler(hub *Hub, pool *pgxpool.Pool) http.HandlerFunc:**
- Get user from auth.UserFromContext(r.Context()) — return 401 if nil
- Call websocket.Accept(w, r, &websocket.AcceptOptions{}) — coder/websocket v1.8.14 API
- Create Client{UserID: user.ID, Conn: conn, Send: make(chan []byte, 256)}
- Send to hub.register
- Defer send to hub.unregister
- Start a write pump goroutine: reads from client.Send, calls conn.Write(ctx, websocket.MessageText, msg)
- Read loop: reads messages from client (for future protocol messages like ack), returns on error to trigger cleanup
- Use r.Context() for the read loop context — when the HTTP request is cancelled, the WS closes

**OpsSinceHandler(pool *pgxpool.Pool) http.HandlerFunc:**
- Get user from auth.UserFromContext(r.Context())
- Parse query param `lamport_ts` as int64 (default 0 if missing)
- Call OpsSince(ctx, pool, user.ID, lamportTS)
- Write JSON array response
