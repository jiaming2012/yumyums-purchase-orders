opsync "github.com/yumyums/hq/internal/sync"

For each handler, add an `opsync.EmitOp(pool, opsync.OpInput{...})` call AFTER the successful write returns and BEFORE writing the HTTP response. The EmitOp call is fire-and-forget (goroutine inside), so it won't block the response.

Per D-02, the frontend does NOT send device_id or lamport_ts yet. During this transitional phase, use `device_id: "server"` and `lamport_ts: 0` for server-emitted ops. The server_ts from the ops table INSERT will provide ordering.

**Handler-by-handler wiring:**

1. **SaveResponseHandler** (line ~321) — after `saveResponse()` returns nil:
   ```go
   opsync.EmitOp(pool, opsync.OpInput{
       DeviceID:   "server",
       UserID:     user.ID,
       EntityID:   body.FieldID,  // the field being saved
       EntityType: "field_response",
       OpType:     opsync.OpSetField,
       Payload:    /* marshal body.Value as JSON */,
       LamportTS:  0,
   })
   ```
