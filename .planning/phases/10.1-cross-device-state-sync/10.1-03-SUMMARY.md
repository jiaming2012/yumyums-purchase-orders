# Phase 10.1 Plan 03: Wire EmitOp into All 7 Workflow Handlers — Summary

**One-liner:** All 7 workflow write handlers now emit fire-and-forget ops via `opsync.EmitOp` after each successful DB write, bridging the existing write paths to the new op-log + pg_notify sync infrastructure.

## What Was Built

Added `opsync "github.com/yumyums/hq/internal/sync"` import to `handler.go` and wired `opsync.EmitOp` calls into all 7 mutation handlers:

| Handler | OpType | EntityType | EntityID |
|---------|--------|------------|----------|
| SaveResponseHandler | `OpSetField` | `field_response` | `input.FieldID` |
| SubmitChecklistHandler | `OpSubmitChecklist` | `submission` | returned submission ID |
| ApproveSubmissionHandler | `OpApproveItem` | `submission` | `body.SubmissionID` |
| RejectItemHandler | `OpRejectItem` | `submission` | `input.SubmissionID` |
| CreateTemplateHandler | `OpSaveTemplate` | `template` | returned template ID |
| UpdateTemplateHandler | `OpSaveTemplate` | `template` | URL param `templateID` |
| ArchiveTemplateHandler | `OpArchiveTemplate` | `template` | URL param `templateID` |

All EmitOp calls are fire-and-forget (runs in goroutine, 5-second timeout). DeviceID is `"server"` and LamportTS is `0` for all server-emitted ops during this transitional phase per D-02. Payload marshaling uses `json.Marshal(map[string]any{...})` — if marshal fails, error is logged and op is skipped (response is not blocked).
