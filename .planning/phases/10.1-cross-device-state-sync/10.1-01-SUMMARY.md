# Phase 10.1 Plan 01: Ops Foundation — Migrations + sync/ops.go

Postgres ops log table, lamport_ts columns on entity tables, and the sync/ops.go package with LWW guard, transactional INSERT+pg_notify, and catch-up query.

## What Was Built

### 0015_ops.sql
Creates the `ops` table — the append-only event log for cross-device sync. Columns: `id` (UUID PK), `device_id` (TEXT), `user_id` (UUID FK), `entity_id` (UUID), `entity_type` (TEXT), `op_type` (TEXT), `payload` (JSONB), `lamport_ts` (BIGINT), `server_ts` (TIMESTAMPTZ), `applied` (BOOLEAN). Two indexes: `ops_user_lamport_idx (user_id, lamport_ts)` for catch-up queries, `ops_entity_idx (entity_id)` for LWW lookups.

### 0016_lamport_ts.sql
Adds `lamport_ts BIGINT NOT NULL DEFAULT 0` to `submission_responses`, `checklist_submissions`, and `checklist_templates`. These columns serve as the LWW guard sentinels — each entity's current winning timestamp lives here.

### backend/internal/sync/ops.go
Complete sync package with:
- **Types:** `OpInput`, `Op`, `ConflictResult`, `ErrConflict`
- **Constants:** `OpSetField`, `OpSubmitChecklist`, `OpApproveItem`, `OpRejectItem`, `OpSaveTemplate`, `OpArchiveTemplate`
- **CheckLWW:** reads entity table `lamport_ts`, implements LWW with device_id tiebreaker for equal timestamps
- **InsertOpAndNotify:** single transaction — CheckLWW + UPDATE entity lamport_ts + INSERT into ops + `pg_notify('ops_channel', ...)`
- **EmitOp:** fire-and-forget goroutine (5-second timeout); ErrConflict logged, not propagated
- **EmitOpWithConflictCheck:** synchronous variant for handlers returning 409
- **OpsSince:** catch-up query with template_assignments subquery for access resolution (D-09)
- **GetOpByID:** single row lookup for PG LISTEN consumer
- **ResolveEntityAccess:** maps entity → template → user IDs for WebSocket fanout
