## Migration Strategy

| Option | Description | Selected |
|--------|-------------|----------|
| Full op-log replacement | Replace autoSaveField/saveResponse with POST /ops. One write path. Rewrites entire data flow. | |
| Layer op-log for sync only | Keep existing writes, add op log as read-only audit trail with WS emission | |

**User's choice:** Incremental migration (Option C)
**Notes:** Gets sync working without rewriting stable code. Each future phase migrates its module naturally.

## WebSocket vs SSE vs Polling

| Option | Description | Selected |
|--------|-------------|----------|
| WebSocket | Bidirectional persistent connection. Go hub + pg_notify. gorilla/websocket or nhooyr.io. | ✓ |
| Server-Sent Events (SSE) | Unidirectional server→client. Native EventSource API. Simpler but one-way. | |
| Polling with catch-up | GET /ops/since every 5-10s. Simplest but battery drain + delay. | |

**User's choice:** WebSocket (Option A)
**Notes:** User asked about iOS Safari killing WS in background. Solution: reconnect-on-visibility + catch-up endpoint. WS for live, catch-up for resume. Same pattern as Notion/Todoist.
