**CheckLWW(ctx, tx, entityID string, entityType string, incomingTS int64, incomingDeviceID string) (*ConflictResult, error):**
Per D-10 and D-12, this function implements the LWW guard:
- Determine the correct table and lookup based on entityType:
  - `"field_response"` -> query `submission_responses` WHERE `field_id = entityID`
  - `"submission"` -> query `checklist_submissions` WHERE `id = entityID`
  - `"template"` -> query `checklist_templates` WHERE `id = entityID`
- SELECT `lamport_ts` FROM the entity table WHERE id/field_id = entityID (within the same transaction)
- If entity not found (no row), return nil (no conflict — first write wins)
- If `incomingTS > currentTS`: no conflict, return nil (incoming wins)
- If `incomingTS == currentTS`: tiebreak by `device_id` lexicographic comparison. If `incomingDeviceID > currentWinnerDeviceID`, no conflict (incoming wins). Otherwise, return ConflictResult with current winner state.
- If `incomingTS < currentTS`: return ConflictResult with current winner's lamport_ts, payload (from the most recent op for this entity), and device_id
- To get the winner's payload: `SELECT payload, device_id FROM ops WHERE entity_id = $1 AND lamport_ts = $2 ORDER BY server_ts DESC LIMIT 1`

**InsertOpAndNotify(ctx, pool, op OpInput) (string, *ConflictResult, error):**
- Begin transaction
- Call CheckLWW(ctx, tx, op.EntityID, op.EntityType, op.LamportTS, op.DeviceID)
- If ConflictResult is non-nil, rollback and return ("", conflictResult, ErrConflict)
- UPDATE the entity table's lamport_ts to op.LamportTS (same entity lookup as CheckLWW)
- INSERT INTO ops (device_id, user_id, entity_id, entity_type, op_type, payload, lamport_ts) VALUES ($1-$7) RETURNING id, server_ts
- SELECT pg_notify('ops_channel', json_build_object('op_id', id, 'entity_id', op.EntityID, 'entity_type', op.EntityType, 'user_id', op.UserID)::text)
- Commit transaction
- Return the op ID, nil, nil

**EmitOp(pool *pgxpool.Pool, op OpInput):**
- Fire-and-forget goroutine (same pattern as cleanupOldDrafts in workflow/handler.go line 254)
- 5-second context timeout
- Calls InsertOpAndNotify
- Logs error on failure, does NOT propagate
- Note: EmitOp is used for server-side emission where conflicts are not expected (the handler already wrote successfully). If InsertOpAndNotify returns ErrConflict, log a warning but do not propagate.

**EmitOpWithConflictCheck(ctx, pool, op OpInput) (string, *ConflictResult, error):**
- Synchronous version of EmitOp that returns conflict information to the caller
- Used by handlers that need to return 409 to the client (future use when frontend sends lamport_ts)
- Simply calls InsertOpAndNotify and returns all three values

**OpsSince(ctx, pool, userID string, lamportTS int64) ([]Op, error):**
- Query: SELECT id, device_id, user_id, entity_id, entity_type, op_type, payload, lamport_ts, server_ts, applied FROM ops WHERE user_id = $1 AND lamport_ts > $2 ORDER BY lamport_ts ASC
- Note: Per D-09, this needs to return ops for entities the user has ACCESS to, not just ops the user created. For Phase 10.1, the access resolution is: query template_assignments to find which template_ids the user is assigned to (as assignee or approver), then return ops where entity_id is one of those template_ids OR where user_id matches.
- The query should be: `SELECT o.* FROM ops o WHERE o.lamport_ts > $2 AND (o.user_id = $1 OR o.entity_id IN (SELECT template_id FROM template_assignments WHERE user_id = $1)) ORDER BY o.lamport_ts ASC`

**GetOpByID(ctx, pool, opID string) (*Op, error):**
- Simple SELECT by primary key, used by the PG listener to fetch full op row after receiving NOTIFY with just the op_id.

**ResolveEntityAccess(ctx, pool, entityID string, entityType string) ([]string, error):**
- Returns list of user IDs who should receive this op via WebSocket
- For entity_type "field_response" or "submission": look up the submission's template_id, then query template_assignments for all assigned user_ids (both assignees and approvers)
- For entity_type "template": query template_assignments for all user_ids assigned to that template
- Returns deduplicated user ID strings
