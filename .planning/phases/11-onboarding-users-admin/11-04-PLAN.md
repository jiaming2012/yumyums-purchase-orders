  <action>
    Create `backend/internal/onboarding/` package with three files.

    **db.go** — SQL query functions:

    1. `GetTemplates(ctx, pool) ([]Template, error)` — SELECT all ob_templates. For each template, fetch sections (ordered by sort_order), and for each section fetch items (ordered by sort_order). For video_series items, fetch video parts (ordered by sort_order). Return nested structure matching what MOCK_OB_TEMPLATES provides.

    2. `GetTemplate(ctx, pool, templateID) (*Template, error)` — Same as above but for a single template.

    3. `GetHireTraining(ctx, pool, hireID, templateID) (*HireTraining, error)` — Returns the template with all sections, items, and video parts, PLUS progress and signoff state for the given hire. Computes section state server-side:
       - Section state = 'signed_off' if ob_signoffs row exists for (section_id, hire_id)
       - Section state = 'complete' if all non-faq items in section are checked (all checkbox items have ob_progress rows, all video_series parts have ob_progress rows)
       - Section state = 'active' if it's the first section or all previous sections are 'complete' or 'signed_off'
       - Section state = 'locked' otherwise
       This mirrors the `isSectionComplete()` + `canUnlockNext()` + `tryAdvanceSections()` logic from onboarding.html (Pitfall 5).

    4. `GetMyTrainings(ctx, pool, hireID) ([]AssignedTemplate, error)` — SELECT ob_template_assignments WHERE hire_id, JOIN ob_templates. For each assigned template, compute overall progress percentage (items_checked / total_items).

    5. `GetManagerHires(ctx, pool) ([]HireOverview, error)` — For admin/manager: SELECT users who have at least one ob_template_assignment, with their assigned template count and overall progress. Used for the Manager tab.

    6. `SaveProgress(ctx, pool, hireID, itemID, progressType string, checked bool) error` — If checked=true: INSERT INTO ob_progress ON CONFLICT DO NOTHING. If checked=false: DELETE FROM ob_progress WHERE hire_id AND item_id AND progress_type. Per D-02: auto-save per item.

    7. `SignOff(ctx, pool, input SignOffInput) error` — INSERT into ob_signoffs with manager_id, section_id, hire_id, notes, rating. ON CONFLICT (section_id, hire_id) DO NOTHING (idempotent). Input struct: `{ManagerID, SectionID, HireID, Notes, Rating string}`.

    8. `CreateTemplate(ctx, pool, input CreateTemplateInput) (string, error)` — INSERT ob_template + sections + items + video_parts in a transaction. Used by Builder tab. Per D-04: full PUT replace via delete+reinsert.

    9. `UpdateTemplate(ctx, pool, templateID string, input CreateTemplateInput) error` — DELETE old sections/items, INSERT new ones in a transaction. Same full-replace pattern as workflow/UpdateTemplate.

    10. `AssignTemplate(ctx, pool, hireID, templateID, assignedBy string) error` — INSERT into ob_template_assignments ON CONFLICT DO NOTHING.

    11. `UnassignTemplate(ctx, pool, hireID, templateID string) error` — DELETE from ob_template_assignments.

    Define structs: `Template`, `Section`, `Item`, `VideoPart`, `HireTraining`, `SectionState`, `AssignedTemplate`, `HireOverview`, `SignOffInput`, `CreateTemplateInput`, `ProgressEntry`.

    **handler.go** — HTTP handlers (RPC-style per D-03):

    Define `writeJSON`, `writeError`, `isAdmin` helpers (same pattern as workflow/handler.go).

    1. `ListTemplatesHandler(pool)` — GET, admin/manager. Returns all templates (for Builder tab and assignment picker).

    2. `GetTemplateHandler(pool)` — GET, authenticated. URLParam "id". Returns single template with full structure.

    3. `MyTrainingsHandler(pool)` — GET, authenticated. Returns current user's assigned templates with progress. Calls GetMyTrainings(ctx, pool, user.ID).

    4. `HireTrainingHandler(pool)` — GET, authenticated. URLParam "hireId" + query "templateId". Returns full training detail with section states for a specific hire+template combo. Used by both the hire (viewing own training) and manager (reviewing hire's progress).

    5. `ManagerHiresHandler(pool)` — GET, admin/manager. Returns list of hires with progress summaries.

    6. `SaveProgressHandler(pool)` — POST, authenticated. Reads `{item_id, progress_type, checked}`. Calls SaveProgress with user.ID as hire_id. Per D-02: immediate auto-save.

    7. `SignOffHandler(pool)` — POST, admin/manager (per D-05). Reads `{section_id, hire_id, notes, rating}`. Validates notes non-empty, rating in ('ready','needs_practice','struggling'). Calls SignOff with user.ID as manager_id.

    8. `CreateTemplateHandler(pool)` — POST, admin/manager. Reads full template structure. Calls CreateTemplate.

    9. `UpdateTemplateHandler(pool)` — PUT, admin/manager. URLParam "id". Reads full template structure. Calls UpdateTemplate. Per D-04: full replace.

    10. `AssignTemplateHandler(pool)` — POST, admin/manager. Reads `{hire_id, template_id}`. Calls AssignTemplate with user.ID as assigned_by.

    11. `UnassignTemplateHandler(pool)` — POST, admin/manager. Reads `{hire_id, template_id}`. Calls UnassignTemplate.

    **seed.go** — Seed data function:
