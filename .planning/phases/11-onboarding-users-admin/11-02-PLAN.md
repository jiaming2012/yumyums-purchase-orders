  <action>
    Create `backend/internal/users/` package with two files.

    **db.go** — SQL query functions using pgxpool.Pool:

    1. `ListUsers(ctx, pool) ([]UserRow, error)` — SELECT all users with derived display_name (same COALESCE expression from Plan 01). Returns: id, email, first_name, last_name, nickname, display_name (derived), role, status, invited_at, accepted_at.

    2. `GetUser(ctx, pool, userID) (*UserRow, error)` — SELECT single user by ID with derived display_name.

    3. `CreateInvitedUser(ctx, pool, input CreateUserInput) (string, error)` — INSERT into users with status='invited', password_hash=NULL, returns user ID. Validate first_name and last_name are non-empty (return error if blank — Pitfall 6). Input struct: `{FirstName, LastName, Email, Role string}`.

    4. `UpdateUser(ctx, pool, userID string, input UpdateUserInput) error` — UPDATE users SET first_name, last_name, nickname, role WHERE id. Input struct: `{FirstName, LastName *string, Nickname *string, Role *string}` (pointer fields for partial update).

    5. `CheckNicknameCollision(ctx, pool, nickname string, excludeUserID string) (string, error)` — Per D-10: query users table to check if `nickname` collides with another user's nickname OR another user's derived display_name. Returns the display_name of the colliding user (empty string if no collision). Use:
       ```sql
       SELECT COALESCE(NULLIF(u.nickname,''), u.first_name || ' ' || LEFT(u.last_name,1) || '.') AS display_name
       FROM users u
       WHERE u.id != $2
         AND (LOWER(u.nickname) = LOWER($1) OR LOWER(COALESCE(NULLIF(u.nickname,''), u.first_name || ' ' || LEFT(u.last_name,1) || '.')) = LOWER($1))
       LIMIT 1
       ```

    6. `InsertInviteToken(ctx, pool, userID, tokenHash string, expiresDays int) error` — INSERT into invite_tokens with expires_at = now() + interval.

    7. `ClaimInviteToken(ctx, pool, tokenHash string) (string, error)` — Atomic UPDATE invite_tokens SET used_at=now() WHERE token_hash=$1 AND used_at IS NULL AND expires_at > now() RETURNING user_id. Returns ErrTokenInvalid if 0 rows.

    8. `ActivateUser(ctx, pool, userID, passwordHash string) error` — UPDATE users SET status='active', password_hash=$2, accepted_at=now() WHERE id=$1.

    9. `DeleteUser(ctx, pool, userID string) error` — DELETE FROM users WHERE id=$1.

    10. `GetAppPermissions(ctx, pool) ([]AppPermission, error)` — SELECT all from hq_apps joined with app_permissions, returning per-app: slug, name, icon, role_grants (array of roles), user_grants (array of user IDs). Group by app.

    11. `SetAppPermissions(ctx, pool, slug string, input SetPermInput) error` — DELETE existing permissions for slug, re-INSERT role_grants and user_grants. Input: `{RoleGrants []string, UserGrants []string}`.

    12. `GetInviteInfo(ctx, pool, tokenHash string) (string, error)` — SELECT u.first_name FROM invite_tokens it JOIN users u ON it.user_id = u.id WHERE it.token_hash = $1 AND it.used_at IS NULL AND it.expires_at > now(). Returns first_name or error.

    Define structs: `UserRow`, `CreateUserInput`, `UpdateUserInput`, `AppPermission`, `SetPermInput`. Define `var ErrTokenInvalid = errors.New("token_invalid")`.

    **handler.go** — HTTP handlers following the RPC pattern from workflow/handler.go:

    Define `writeJSON`, `writeError`, `isAdmin` helpers (same as workflow/handler.go).

    1. `ListUsersHandler(pool)` — GET, admin-only. Calls ListUsers, returns JSON array.

    2. `InviteHandler(pool)` — POST, admin-only (per D-12). Reads `{first_name, last_name, email, role}`. Validates non-empty first_name, last_name, email. Calls CreateInvitedUser, then auth.GenerateToken(), InsertInviteToken(7 days), returns `{user: UserRow, invite_path: "/login.html?token=<raw>"}`. Per D-14: opaque token, no email sent.

    3. `UpdateUserHandler(pool)` — PATCH, admin-only. Reads chi URLParam "id". Reads `{first_name, last_name, nickname, role}`. If nickname is non-empty, calls CheckNicknameCollision — on collision returns 409 with `{"error": "nickname_taken", "message": "'X' is already taken by Y."}` (per D-11). Calls UpdateUser.

    4. `ResetPasswordHandler(pool)` — POST, admin-only (per D-19). Reads chi URLParam "id". Calls auth.GenerateToken(), InsertInviteToken(7 days), returns `{reset_path: "/login.html?token=<raw>"}`.

    5. `RevokeHandler(pool)` — POST, admin-only (per D-20). Reads chi URLParam "id". Calls auth.DeleteAllSessionsByUserID. Returns 204.

    6. `DeleteUserHandler(pool)` — DELETE, admin-only. Reads chi URLParam "id". Calls DeleteUser. Returns 204.

    7. `InviteInfoHandler(pool)` — GET, **unauthenticated** (per RESEARCH). Reads `token` query param. Calls auth.HashToken(token), GetInviteInfo. Returns `{first_name}` or 400 token_expired.

    8. `AcceptInviteHandler(pool, secureCookie bool)` — POST, **unauthenticated** (per RESEARCH). Reads `{token, password}`. Calls auth.HashToken(token), ClaimInviteToken (atomic — Pitfall 3), auth.HashPassword(password), ActivateUser, auth.CreateSession. Sets hq_session cookie (replicate LoginHandler cookie pattern — same MaxAge, Path, HttpOnly, Secure, SameSite). Returns `{user: UserRow}` same shape as login response. Per D-17: status changes to 'active'.

    9. `GetAppPermissionsHandler(pool)` — GET, admin-only. Calls GetAppPermissions.

    10. `SetAppPermissionsHandler(pool)` — PUT, admin-only. Reads chi URLParam "slug". Reads `{role_grants, user_grants}`. Calls SetAppPermissions.

    Import `auth` package as `"github.com/jiaming2012/yumyums-hq/internal/auth"`. Match the existing module path from go.mod.
