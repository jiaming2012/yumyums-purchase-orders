    Create backend/internal/receipt/worker.go:
    - StartWorker(ctx context.Context, cfg WorkerConfig) — starts a time.Ticker goroutine
    - Interval from RECEIPT_WORKER_INTERVAL env var (default "6h" per research recommendation)
    - If MercuryAPIKey or AnthropicAPIKey is empty: log "receipt worker: skipping — missing API keys" and return without starting ticker (graceful skip per D-10)
    - runIngestCycle(ctx, cfg) error — the main pipeline loop:
      1. Compute date range: now minus LookbackDays (default 14 for ongoing, MERCURY_LOOKBACK_DAYS env var)
      2. FetchTransactions(ctx, apiKey, startDate, endDate)
      3. For each transaction with attachments:
         a. Check if bank_tx_id already exists in purchase_events OR pending_purchases (skip if so — idempotent)
         b. Download receipt file from attachment URL
         c. Optionally upload original to DO Spaces: receipts/{bank_tx_id}/original.{ext} (if SpacesPresigner configured)
         d. ParseReceipt with Claude Haiku
         e. ValidateReceiptData against bank amount
         f. If valid:
            - Load existing purchase_items map from DB
            - For each item: DerivePurchaseItemID (fuzzy match or create new)
            - Auto-create vendor if not exists (INSERT ON CONFLICT DO NOTHING, per D-09)
            - Insert purchase_event + purchase_line_items in a transaction
         g. If invalid:
            - Insert into pending_purchases with reason, items as JSONB, receipt_url
            - Log "receipt worker: transaction {id} routed to review queue: {reason}"
      4. Log summary: "receipt worker: processed {N} transactions, {M} auto-created, {K} pending review"

    Wire in main.go:
    - Read env vars: MERCURY_API_KEY, ANTHROPIC_API_KEY, RECEIPT_WORKER_INTERVAL, MERCURY_LOOKBACK_DAYS
    - Build WorkerConfig
    - Call receipt.StartWorker(ctx, cfg) after server routes are registered (but before ListenAndServe)
    - Pass the same context that the server uses so worker shuts down on SIGTERM
