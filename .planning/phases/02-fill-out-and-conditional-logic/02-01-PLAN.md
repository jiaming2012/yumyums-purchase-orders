**1. Constants and State (per D-07):**

```javascript
const CURRENT_USER = { name: 'Jamal M.', initials: 'JM' };
const DAY_NAMES = ['Sunday','Monday','Tuesday','Wednesday','Thursday','Friday','Saturday'];

let fillState = {
  fillView: 'list',         // 'list' | 'runner'
  activeTemplateId: null,
  responses: {}              // keyed by templateId
};
```

**2. Pure Functions (no DOM reads, no state mutations):**

- `evaluateCondition(condition, completedItems)` — handles null (return true), condition.days (check today), condition.field_id + operator (equals/not_equals/lt/gt/in_range). Single-pass, no loops. See 02-RESEARCH.md Pattern 3.
- `isSectionVisible(sec)` — checks sec.condition.days against today. No condition = visible.
- `isFieldVisible(fld, completedItems)` — delegates to evaluateCondition(fld.condition, completedItems).
- `evaluateFailTrigger(fail_trigger, value)` — returns false for empty/null/undefined value. Handles out_of_range, equals, not_equals types. (Used in Plan 02 but define now.)
- `getVisibleFields(template, completedItems)` — returns flat array of {field, sectionId} for visible, non-photo fields.
- `computeProgress(template, completedItems)` — iterates sections (skip invisible), iterates fields (skip invisible + photo), counts total and completed. Returns `{ total, completed }`.
- `formatTime(isoString)` — returns "H:MMa/p" format (e.g., "2:15p"). 12-hour, no leading zero on hour.
- `clearHiddenFieldAnswers(tplId)` — iterates template sections/fields, deletes completedItems and failNotes entries for invisible fields. Called before every render.

**3. Mutation Functions:**

- `openChecklist(tplId)` — sets `fillState.fillView = 'runner'`, `fillState.activeTemplateId = tplId`. Creates response object in `fillState.responses[tplId]` if not exists: `{ templateId, startedAt: new Date().toISOString(), completedItems: {}, failNotes: {}, submitted: false }`. Calls `renderFillout()`.
- `backToFillList()` — sets `fillState.fillView = 'list'`, `fillState.activeTemplateId = null`. Calls `renderFillout()`. Does NOT clear response state.
- `toggleCheckItem(tplId, fldId)` — toggles completedItems[fldId]: if exists, delete; if not, set `{ value: true, answeredBy: CURRENT_USER, answeredAt: new Date().toISOString() }`. Calls `renderFillout()`.

**4. Render Functions:**

- `renderFillout()` — gets `#s1` element. If runner view, calls `clearHiddenFieldAnswers()` first. Sets innerHTML to `renderFillList()` or `renderRunner()`.
- `renderFillList()` — filters MOCK_TEMPLATES by day: `!t.active_days || t.active_days.length === 0 || t.active_days.includes(new Date().getDay())`. If empty, render empty state (per D-09): heading "No checklists for today." and body "Enjoy your day off, {DAY_NAMES[today]}!" using `.empty-state` pattern with 32px vertical padding, centered text.
  If templates exist, render `<div class="cat">Today's Checklists</div>` followed by cards.
- `renderChecklistCard(tpl)` — card with: template name at 15px/500, section count "{N} sections" at 12px var(--mut), approval badge if `tpl.requires_approval` using warn-bg/warn-tx, progress bar (4px track with var(--brd) background, fill with var(--info-tx)). Whole card is tappable: `data-action="open-checklist" data-tpl-id="${tpl.id}"`. Card uses existing `.card` class + margin-bottom 8px.
- `renderRunner(tpl)` — back link "‹ Checklists" with `data-action="back-to-list"` using `.editor-back` class. Progress card showing "{completed} of {total} items complete" at 12px var(--mut). Then iterates visible sections: `.cat` header (sticky) + fields. Only renders checkbox fields in this task (yes_no, text, temperature stubs come in Plan 02). Submit button placeholder at bottom.
- `renderFillField(fld, tplId, answer, failNote)` — switch on fld.type. For `checkbox`: renders `.fill-row` with `data-action="toggle-check"`, min-height 44px, flex row with label (14px, strikethrough + var(--mut) if done) and right side (green "✓" at 16px + `.attr-badge` "{initials} . {time}" at 12px var(--mut) if done, else "○" circle at 16px var(--mut)). For `photo`: return empty string (skipped in Phase 2). For other types: render a placeholder `.fill-row` showing fld.label + " (coming soon)" — Plan 02 will replace these.

**5. Event Delegation:**

Attach ONE click listener and ONE input listener on `document.getElementById('s1')` at script boot (outside any function). Use `e.target.closest('[data-action]')?.dataset.action` routing pattern. Click actions for this task: `open-checklist`, `back-to-list`, `toggle-check`.
