4. **Pure functions:**
   - `generateId(prefix)` — returns `prefix + '_' + Date.now() + '_' + Math.random().toString(36).slice(2,6)`
   - `getDefaultConfig(type)` — returns config defaults: temperature `{unit:'°F',min:0,max:500}`, photo `{}`, others `{}`
   - `createNewTemplate()` — returns template object with `generateId('tpl')`, empty name, empty sections array, `active_days: null`, `requires_approval: false`
   - `createNewSection(title)` — returns section with `generateId('sec')`, title, order, empty fields, `condition: null`
   - `createNewField(type)` — returns field with `generateId('fld')`, type, empty label, `required: false`, `order: 0`, config from `getDefaultConfig(type)`, null fail_trigger/corrective_action/condition. For temperature type, also populate `fail_trigger: {type:'out_of_range', min:0, max:500}` and `corrective_action: {label:'Corrective action taken', type:'text', required:true}`

5. **Render functions:**
   - `renderBuilder()` — clears `#builder-body` innerHTML. If `state.view === 'list'`, calls `renderTemplateList()`. If `'editor'`, calls `renderEditor()`.
   - `renderTemplateList()` — returns HTML string: `.cat` header "TEMPLATES", a `.card` with one `.row` per template (showing `.nm` name and `.mt` section count), and a `.btn-primary` "+ New template" button. If no templates, show `.empty-state` with heading "No templates yet" and body "Create your first checklist template to get started."
   - `renderEditor()` — returns HTML string: `.editor-back` link "‹ Templates" (onclick `backToList()`), `.meta-card` with `.hd` containing template name `.field input` and `.toggle-row` for requires-approval toggle (reuse `.toggle` + `.slider` pattern from users.html), then loop through `state.activeTemplate.sections` calling `renderSection(sec)`, then `.add-section-btn` "+ Add section", then `.btn-primary` "Save template".
   - `renderSection(sec)` — returns HTML: `.sec-hd` with `sec.title` and `.sec-delete` button (aria-label="Delete section"), `.field-list[data-section-id]` container with `sec.fields.map(f => renderField(f, sec.id)).join('')`, `.add-field-btn` "+ Add field", `.section-divider`

6. **Mutation functions (all follow state -> renderBuilder() pattern):**
   - `openEditor(tplId)` — finds template by id OR creates new one via `createNewTemplate()` + pushes to `MOCK_TEMPLATES`, sets `state.activeTemplate`, `state.view = 'editor'`, calls `renderBuilder()`
   - `backToList()` — resets state.view to 'list', nulls activeTemplate/expandedField/fieldPickerSection, calls `renderBuilder()`
   - `addSection()` — prompts title with `prompt('Section name:')`, if non-empty creates section via `createNewSection()`, pushes to `state.activeTemplate.sections`, calls `renderBuilder()`
   - `deleteSection(secId)` — filters section from `state.activeTemplate.sections`, calls `renderBuilder()`
   - `updateTemplateName(name)` — sets `state.activeTemplate.name = name` (no re-render needed, just state sync)
   - `toggleApproval()` — flips `state.activeTemplate.requires_approval`
