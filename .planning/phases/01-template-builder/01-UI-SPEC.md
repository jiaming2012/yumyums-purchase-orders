## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tab bar inner padding, chip gaps |
| sm | 8px | Card margin-bottom, row gap between sections |
| md | 12px | Default row padding vertical, page top padding, card border-radius gaps |
| md+ | 16px | Row padding horizontal, section header padding, field input padding horizontal |
| lg | 24px | Page-level section breaks |
| xl | 32px | Not currently used in existing pages — available |

Exceptions:
- Body horizontal padding: 12px (purchasing.html convention; not a standard token multiple but established project pattern)
- Tab pill padding: 10px vertical (existing `.tabs button` pattern)
- Touch targets: minimum 44px tall for all tappable field rows and day-of-week chips (FILL-03 mobile requirement; enforced via `min-height: 44px`)
- Drag handle touch area: 44px wide × 44px tall minimum on the left side of each field row

Source: measured from `purchasing.html` and `users.html` CSS.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / field label | 14px | 400 (regular) | 1.4 |
| Small / meta / subtext | 11–12px | 400 (regular) | 1.4 |
| UI label (form labels, section category) | 12px | 500 (medium) | 1.0 (single line) |
| Heading / card title | 15px | 500 (medium) | 1.2 |

Notes:
- Only 2 weights used across the entire codebase: 400 (regular) and 500 (medium). The exception is `.btn-primary` at 600 (semibold) — reserved for primary action buttons only.
- Font sizes in use: 11px (micro labels, pills), 12px (subtext, form field labels), 13px (secondary body), 14px (primary body, field labels), 15px (card headings). Use 14px as default body; 12px for supporting text; 15px for the template name heading.
- Section category headers (`.cat` pattern): 11px, weight 500, uppercase, letter-spacing 0.5px, `var(--mut)` color — use this for "SECTIONS" divider labels.

Source: measured from `purchasing.html` and `users.html` CSS.

| Destructive | `#c0392b` | `#c0392b` | hardcoded (existing `.btn-danger` pattern) | Delete field, delete section — border + text only, no fill |

Accent (`var(--info-bg)` / `var(--info-tx)`) reserved for:
1. Active tab indicator (`.tabs button.on`)
2. Active day-of-week chips (`.day-chip.on`)
3. "Save template" primary CTA button background
4. Focus ring on text inputs (border-color: `var(--info-tx)`)
5. Back link color and active toggle/slider fill
6. Field type picker "confirm" button background

Accent is NOT used for: general field rows, section headers, the drag handle, inactive states, or decorative elements.

## Interaction Contracts

### Field Row Expand/Collapse

- **Trigger:** Tap anywhere on the field label area (`.field-row-tap` target, excludes drag handle and delete button)
- **Behavior:** One field expanded at a time (`state.expandedField`). Tapping the active field collapses it. Tapping another field collapses the current and expands the new one.
- **Animation:** None — instant show/hide (matches existing page pattern; no CSS transitions used in codebase)
- **Expanded panel contents vary by field type:**
  - All types: label text input, day-of-week chips, skip logic dropdowns (if preceding fields exist)
  - `temperature` only: min input + max input + `°F` label
  - `photo` only: "Required" toggle switch
  - `checkbox`, `yes_no`, `text`: no type-specific extras beyond common controls

### Day-of-Week Chips

- **Labels:** M T W T F Sa Su (two-letter for Sa/Su to avoid T/T ambiguity — confirmed in CONTEXT.md specifics)
- **Default state:** All 7 chips inactive = no day restriction = show every day
- **Active state:** `var(--info-bg)` background, `var(--info-tx)` text, weight 500
- **Tap behavior:** Toggle individual day in/out of `active_days` array; immediate state update + re-render
- **Storage:** `active_days` as int array (0=Sun, 1=Mon…6=Sat); null condition = show all days

### Field Type Picker

- **Trigger:** Tap "+ Add field" button at bottom of a section
- **Appearance:** Inline card expanding below the button — NOT a modal or popover. 5 rows, one per type.
- **Field types with display labels:**
  - `checkbox` — "Checkbox"
  - `yes_no` — "Yes / No"
  - `text` — "Text note"
  - `temperature` — "Temperature (°F)"
  - `photo` — "Photo"
- **Dismiss:** Tap a type row (adds field + closes picker) or tap elsewhere (closes picker, no field added)
- **State:** `state.fieldPickerSection` tracks which section's picker is open; null = all closed

### Drag to Reorder

- **Handle:** `☰` glyph on left of field row; `.drag-handle` class; 44px × 44px minimum touch target
- **Library:** SortableJS 1.15.7 via `https://unpkg.com/sortablejs@1.15.7/Sortable.min.js`
- **Animation:** 150ms (SortableJS `animation` option)
- **Scope:** Within a section only. Cross-section drag is not supported in Phase 1.
- **After drag:** Sync new order to `state.activeTemplate` via `onEnd` handler; do NOT trigger full re-render (SortableJS already moved the DOM node)

### Builder Tab Access Gate

- **Check:** `MOCK_CURRENT_USER.permissions.includes('workflow_builder')`
- **If no access:** Tab 2 button is not rendered at all (not just hidden with CSS)
- **Mock default:** Admin and manager roles have builder permission; crew/team_member does not

### Template List to Editor Navigation

- **Sub-view state:** `state.view` = `'list'` or `'editor'`
- **List view:** Shows all templates in `MOCK_TEMPLATES` as `.row` items + "New template" button
- **Editor view:** Shows template name input + requires-approval toggle + sections + fields
- **Back from editor:** "‹ Templates" link above the editor — uses `backToList()`, resets `state.expandedField`

### Requires Approval Toggle

- **Placement:** Template meta card at the top of the editor view, below template name input, above first section. Single `.card` with `.hd` containing template name input and a `.toggle-row` for the requires-approval toggle.
- **Copy:** "Requires approval" (label), "Manager must approve before checklist is considered complete" (sub-label using `.sub` class)
- **Component:** Reuse `.toggle` + `.slider` CSS from `users.html`

## Page Layout Contract

```
workflows.html
  body (bg: var(--bg), padding: safe-area-inset + 12px horizontal)
    .app (max-width: 480px, centered)
      .back               "‹ HQ"  → index.html
      .tabs               [Fill Out] [Builder]
      #s1                 fill-out tab — stubbed in Phase 1 (empty placeholder)
      #s2                 builder tab
        [list view]
          .cat            "TEMPLATES" section label
          .card
            .row*         one row per template
          .btn-primary    "+ New template" (full width, bottom of list)

        [editor view]
          .back (inline)  "‹ Templates" → backToList()
          .card           template meta card
            .hd
              .field      Template name input
            .toggle-row   Requires approval toggle
          [sections loop]
            .cat          section title (uppercase, sticky)
            .field-list   SortableJS container (data-section-id)
              .field-row* one per field
                .drag-handle  ☰
                .field-row-tap  field label (tap to expand)
                .pill       field type badge
                .field-delete ✕ button
                [if expanded]
                .field-settings
                  .field   Label input
                  [day chips row]
                  [type-specific controls]
                  [skip logic row, if preceding fields exist]
            .add-field-btn  "+ Add field" (per section)
            [if picker open] .field-type-picker card
            .section-divider  0.5px var(--brd) horizontal rule
          .btn secondary  "+ Add section"
          .btn-primary    "Save template"
```
