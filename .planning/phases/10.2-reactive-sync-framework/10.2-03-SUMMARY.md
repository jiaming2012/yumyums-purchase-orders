**One-liner:** POST /ops unified write endpoint with dependency-injection OpRouter, submitOp migrated to send real device_id + lamport_ts, _recentSaves timing hack eliminated.

## What Was Built

### Backend: POST /ops Endpoint

- `OpRequest` struct with `op_type`, `entity_id`, `entity_type`, `payload`, `lamport_ts`, `device_id`
- `OpHandler(pool, router)` — routes ops to business logic via injected `OpRouter` function, then calls `InsertOpAndNotify` with client's real `device_id` and `lamport_ts`
- `OpRouterError` — carries HTTP status + message through function return for structured error responses
- `RouteOpResult` — carries updated `entity_id` back from router (e.g. new submission ID)
- `workflowOpRouter(pool)` in `main.go` — wires both packages, handles routing for all 6 op types: `SET_FIELD`, `SUBMIT_CHECKLIST`, `APPROVE_ITEM`, `REJECT_ITEM`, `SAVE_TEMPLATE`, `ARCHIVE_TEMPLATE`
- Exported var aliases in `workflow/handler.go`: `SaveResponseFunc`, `SubmitChecklistFunc`, `ValidateFailNotesFunc`, `ApproveSubmissionFunc`, `RejectItemFunc`, `CreateTemplateFunc`, `UpdateTemplateFunc`, `ArchiveTemplateFunc`
- Route registered in `main.go`: `r.Post("/ops", opsync.OpHandler(pool, workflowOpRouter(pool)))`

### Frontend: submitOp Migration

- `submitOp` rewritten to call `api('POST', 'ops', {...})` with `op_type`, `entity_id`, `entity_type`, `payload`, `lamport_ts` (from `LAMPORT_CLOCK.tick()`), `device_id` (from `LAMPORT_CLOCK.deviceId`)
- `_recentSaves` const removed from sync.js — no longer needed
- `_recentSaves[field_id]` check removed from `applyOp` — self-echo handled by `op.device_id === LAMPORT_CLOCK.deviceId`
- `window._recentSaves` export removed
- `autoSaveField` function removed from sync.js (was kept for backward compat, but workflows.html never used it — uses `debouncedSaveField` instead)
- `_recentSaves[fieldId] = Date.now()` lines removed from `debouncedSaveField` in `workflows.html`
- SW rebuilt with new content hash for sync.js

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Architecture] Circular import: sync → workflow → sync**
- **Found during:** Task 1
- **Issue:** Plan spec had `OpHandler` directly import the `workflow` package. But `workflow/handler.go` already imports `sync` (for `EmitOp`). `sync` importing `workflow` creates a circular dependency.
- **Fix:** Introduced `OpRouter` function type (dependency injection). `OpHandler` accepts an `OpRouter` parameter. `workflowOpRouter(pool)` in `main.go` provides the concrete routing implementation. `main.go` imports both packages — the natural wiring point.
- **Files modified:** `backend/internal/sync/handler.go`, `backend/cmd/server/main.go`
- **Commit:** 165127d
