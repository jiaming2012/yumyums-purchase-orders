# Phase 10.2 Plan 02: Wire workflows.html to sync.js Summary

**workflows.html refactored to use sync.js Store for all state — globals replaced by store collections, autoSaveField replaced by debouncedSaveField via submitOp, 5 store.on subscribers drive reactive re-renders**

## Performance

- **Duration:** ~30 min

- **Completed:** 2026-04-17T15:02:00Z
- **Tasks:** 1 of 1
- **Files modified:** 3 (workflows.html, sync.js, sw.js)

## Accomplishments
- Removed ~450 lines of duplicated code from workflows.html (api, getDB, LamportClock, wsConnect, autoSaveField, etc.) — all now provided by sync.js
- All 7 global state collections registered in store and aliased for backward compatibility
- 14 autoSaveField call sites replaced by debouncedSaveField using submitOp (D-08 single write channel)
- 5 store.on subscribers registered for reactive rendering (myChecklists, mySubmissions, draftResponses, pending, templates)
- store.batch used in loadMyChecklists for atomic triple-collection update
- applyOp in sync.js updated to use store.set/delete instead of directly mutating FIELD_RESPONSES/DRAFT_RESPONSES globals

## Task Commits

1. **Task 1: Wire workflows.html to sync.js** - `f93481e` (feat)

## Files Created/Modified
- `workflows.html` - Removed extracted code, added sync.js script tag, store.register, store.on subscribers, debouncedSaveField, store.batch in loadMyChecklists
- `sync.js` - Updated applyOp to use store.set/delete, exposed _recentSaves on window
- `sw.js` - Rebuilt by Workbox after workflows.html + sync.js changes (content-hash update)

## Decisions Made
- `debouncedSaveField` adds `_recentSaves[fieldId]` tracking after successful save (same as old `autoSaveField`) — LAMPORT_CLOCK device_id check alone is insufficient when the clock hasn't been initialized before the first field click
- Kept explicit `renderMyChecklists()` call in `loadMyChecklists` after `hydrateFieldState()` — the `store.on('myChecklists')` subscriber fires before `hydrateFieldState()` runs, so FIELD_RESPONSES is stale at subscriber fire time, causing wrong progress counts and missing submission badges
- Used `store.setAll('pending', ...)` for inline PENDING mutations (approve/reject handlers) to keep alias and store in sync

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] debouncedSaveField missing _recentSaves echo suppression**
- **Found during:** Task 1 (running sync tests)
- **Issue:** The plan spec for `debouncedSaveField` omitted `_recentSaves` tracking, causing the "no updated-by toast for own saves" test to fail — LAMPORT_CLOCK deviceId check fails when clock is null during first save
- **Fix:** Added `_recentSaves[fieldId] = Date.now()` after successful save in `debouncedSaveField`; exposed `_recentSaves` on `window` from sync.js; re-added `_recentSaves[field_id]` guard in `applyOp`
- **Files modified:** workflows.html, sync.js
- **Verification:** `tests/sync.spec.js:246` (no toast for own saves) now passes
- **Committed in:** f93481e (Task 1 commit)

**2. [Rule 1 - Bug] store.batch fires subscriber before hydrateFieldState — submission badges missing**
- **Found during:** Task 1 (running persistence tests)
- **Issue:** `store.on('myChecklists')` fires during batch completion (synchronously) before `hydrateFieldState()` runs. `renderChecklistList` uses `MY_SUBMISSIONS` to render Pending/Submitted badges, but `MY_SUBMISSIONS` alias wasn't updated yet when subscriber ran
- **Fix:** Added explicit `renderMyChecklists()` call after `hydrateFieldState()` in `loadMyChecklists` when not in runner view
- **Files modified:** workflows.html
- **Verification:** `tests/persistence.spec.js:406` (submitted checklist shows submitted after reload) now passes
- **Committed in:** f93481e (Task 1 commit)

**3. [Rule 1 - Bug] Inline PENDING mutations (approve/reject handlers) bypassed store**
- **Found during:** Task 1 (code review)
- **Issue:** Two event handlers did `PENDING = PENDING.filter(...)` which updated local alias but left store stale
- **Fix:** Changed to `store.setAll('pending', PENDING.filter(...)); PENDING = store.get('pending')`
- **Files modified:** workflows.html
- **Verification:** Store stays in sync with local alias after approve/reject actions
- **Committed in:** f93481e (Task 1 commit)
