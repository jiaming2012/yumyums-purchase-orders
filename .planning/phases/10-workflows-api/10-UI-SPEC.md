## Spacing Scale

Declared values (multiples of 4 only). Source: extracted from workflows.html CSS patterns.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tab padding (`.tabs` padding: 4px), inline gap between icons and text |
| sm | 8px | Card margin-bottom (`.card` margin-bottom: 8px), compact row gaps |
| md | 12px | Body horizontal page padding, back-link margin-bottom |
| lg | 16px | Section header padding (`.cat` padding: 6px 16px), field row horizontal padding |
| xl | 24px | Major content padding (`.empty-state` padding: 32px 16px) |
| 2xl | 32px | Empty state top/bottom padding |
| touch | 44px | Minimum touch target height for all interactive controls |

Exceptions:
- Body padding uses `env(safe-area-inset-top)` and `env(safe-area-inset-bottom)` for iOS safe area — do not remove these
- Drag handle minimum: 44×44px (`min-width:44px;min-height:44px`) — iOS touch target requirement
- Tabs button padding: `12px 16px` (12px vertical is the `md` token — on the 4-point grid)

## Typography

Source: extracted from workflows.html CSS. All weights are system font stack weights.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label / meta | 12px | 500 (medium) | 1.4 |
| Section heading | 15px | 500 (medium) | 1.2 |
| Display / tab | 13px | 500 (medium) | 1.0 (single-line UI elements) |

Notes:
- Uppercase tracking labels (`.cat`, `.field label`): `text-transform: uppercase; letter-spacing: 0.5px` at 12px weight 500 — preserve this pattern for all new category/field labels
- Attribution / timestamp lines (`.fill-attribution`, `.sub`): 12px weight 400, `color: var(--mut)` — use for auto-save timestamp and user attribution text; de-emphasis is achieved through muted color, not a separate font size
- No heading larger than 15px — this is a mobile-first dense UI, not a marketing page

## New UI Patterns (Phase 10 additions)

These patterns are new to this phase. All must use existing CSS variable tokens.

### Skeleton Screens (D-24)

Placeholder shapes shown while API data loads. Replaces spinners entirely.

```
Structure: div.skeleton — rounded rect placeholder shape
CSS:
  background: linear-gradient(90deg, var(--card) 25%, var(--brd-solid) 50%, var(--card) 75%)
  background-size: 200% 100%
  animation: skeleton-pulse 1.4s ease infinite
  border-radius: 6px

@keyframes skeleton-pulse {
  0% { background-position: 200% 0 }
  100% { background-position: -200% 0 }
}
```

Skeleton shapes match content layout:
- Template list item: full-width bar 15px tall, 60% width bar 12px tall below it
- Checklist row: full-width 14px bar, 40% 12px bar
- Approvals group heading: 30% 12px bar
- All skeletons: margin and padding matching the real content row dimensions

### Pending Sync Banner (D-13)

Persistent top banner when IndexedDB queue has items. Positioned below the tab bar, above the list content.

```
.sync-banner
  background: var(--warn-bg)
  color: var(--warn-tx)
  font-size: 13px
  font-weight: 500
  padding: 8px 16px
  text-align: center
  border-radius: 8px
  margin-bottom: 8px
```

Copy: "N submission(s) pending sync" (N = queue count). Clears automatically when queue drains.

Per-item pending badge on queued checklist rows:

  background: var(--warn-bg)
  color: var(--warn-tx)
  font-size: 12px
  font-weight: 500
  margin-left: 8px

Copy: "Pending sync"

Per-field auto-save error indicator: small red dot (4px circle, `background: #c0392b`) at trailing edge of field row. Tap the dot to retry.

## Interaction Contracts

### Auto-save (D-21)

- Trigger: Each field tap (checkbox), blur (text inputs), or select change
- Debounce: 400ms after last input event before API call fires
- Visual feedback: Per-field "Saved" micro-indicator fades in then out (500ms visible, 300ms fade)
- Error: Red dot at field trailing edge on failure; tapping dot retries immediately

### Builder explicit save (D-08)

- "Save checklist" button: enabled only when template has at least one section and one field
- Disabled state: `opacity: 0.5; pointer-events: none`
- Saving state: button text changes to "Saving…" (no spinner), re-enables on response
- "Discard changes" button: shows confirm prompt inline ("Discard unsaved changes? [Discard] [Keep editing]") — not a browser `confirm()` dialog

### Submit checklist

- "Submit checklist" button: always visible at bottom of fill view; tapping it validates required fields first
- Validation error: inline message below button — "Complete all required fields before submitting." — no modal
- Submission in flight: button text "Submitting…", disabled
- Success: navigate back to My Checklists tab, brief toast "Submitted ✓"
- Offline path: queued silently, toast "Queued for sync when back online", sync banner appears

### Approval flow

- Manager taps submission in Approvals tab → opens per-field review view
- Each field row has "Reject" inline button (`.review-reject-btn`)
- Tapping "Reject" expands rejection reason textarea inline below the field row (no modal)
- "Approve submission" button at bottom is disabled until manager has reviewed all flagged items or no items are flagged
- "Confirm rejection" button text: "Send rejection" (not "Confirm")
- × delete icons must include `aria-label="Delete"` (or context-appropriate label) for screen reader accessibility

### Offline → online transition

- `online` event fires → IndexedDB queue drains sequentially (one request at a time, not parallel)
- Each successful sync: removes item from queue, decrements banner count
- Banner disappears when count reaches 0 (no explicit dismiss needed)
- 409 conflict: stops drain for that item, shows inline error card in My Checklists tab

## States Inventory

Every dynamic surface in workflows.html must handle these states:

| Surface | States |
|---------|--------|
| My Checklists tab | loading (skeleton), empty (today, no checklists), populated, offline-queued |

| Builder template list | loading (skeleton), empty (no templates), populated, admin-restricted |
| Builder editor | pristine, dirty (unsaved changes), saving, save-error |
| Approvals tab | loading (skeleton), empty (caught up), populated |
| Approval review | reviewing, approving, rejecting, submitting |
| Sync banner | hidden (queue empty), visible (N items) |
| Per-field auto-save | idle, saving, saved, error |
