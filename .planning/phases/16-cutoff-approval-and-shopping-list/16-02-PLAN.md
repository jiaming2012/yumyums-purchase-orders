    **GetCutoffConfig(ctx, pool)** -- SELECT single row from cutoff_config. Returns nil if no config set (not an error).

    **UpsertCutoffConfig(ctx, pool, dayOfWeek int, cutoffTime string, timezone string)** -- INSERT INTO cutoff_config ON CONFLICT (id) DO UPDATE. Since there's only one row, use a fixed approach: DELETE all rows then INSERT, or use a known ID. Simplest: DELETE FROM cutoff_config; INSERT INTO cutoff_config (...). This is safe because it's a single-row table. Return the new CutoffConfig.

    **LockPO(ctx, pool, poID string)** (per D-14, Pitfall 2):
    - BEGIN TX
    - `UPDATE purchase_orders SET status = 'locked', locked_at = now(), version = version + 1 WHERE id = $1 AND status = 'draft' RETURNING id`
    - If 0 rows: return ErrPONotDraft
    - Get the PO's week_start, compute next Monday (+7 days)
    - `INSERT INTO purchase_orders (week_start, status) VALUES ($nextWeek, 'draft') ON CONFLICT (week_start) DO NOTHING`
    - COMMIT
    - Return nil

    **UnlockPO(ctx, pool, poID string)** (per D-13):
    - Check PO status first. If status is 'approved', 'shopping_active', or 'completed': return ErrUnlockAfterApproval
    - `UPDATE purchase_orders SET status = 'draft', locked_at = NULL, version = version + 1 WHERE id = $1 AND status = 'locked' RETURNING id`
    - If 0 rows: return ErrPONotLocked
    - Return nil

    **ApprovePO(ctx, pool, poID string, approvedBy string)** (per D-10, D-11, D-15, Pattern 2):
    - BEGIN TX
    - Check no active shopping list exists: `SELECT COUNT(*) FROM shopping_lists WHERE status = 'active'`. If count > 0: return ErrActiveShoppingListExists
    - `UPDATE purchase_orders SET status = 'shopping_active', approved_at = now(), approved_by = $2, version = version + 1 WHERE id = $1 AND status = 'locked' RETURNING id, week_start`
    - If 0 rows: return ErrPONotLocked
    - Create shopping list: `INSERT INTO shopping_lists (po_id) VALUES ($1) RETURNING id`
    - Get distinct vendors from PO line items (JOIN po_line_items with purchase_items for vendor_id, JOIN vendors for name):
      ```sql
      SELECT DISTINCT COALESCE(pi.vendor_id::text, 'unassigned') as vid, COALESCE(v.name, 'Unassigned') as vname
      FROM po_line_items pli
      JOIN purchase_items pi ON pi.id = pli.purchase_item_id
      LEFT JOIN vendors v ON v.id = pi.vendor_id
      WHERE pli.po_id = $1
      ```
    - For each vendor: INSERT INTO shopping_list_vendor_sections (shopping_list_id, vendor_id, vendor_name)
    - Snapshot items grouped by vendor section:
      ```sql
      INSERT INTO shopping_list_items (shopping_list_id, vendor_section_id, purchase_item_id, item_name, photo_url, store_location, quantity, unit)
      SELECT $listID, $sectionID, pi.id, COALESCE(pi.full_name, pi.description), pi.photo_url, pi.store_location, pli.quantity, pli.unit
      FROM po_line_items pli
      JOIN purchase_items pi ON pi.id = pli.purchase_item_id
      WHERE pli.po_id = $poID AND COALESCE(pi.vendor_id::text, 'unassigned') = $vendorKey
      ```
    - COMMIT
    - Return the shopping list ID

    SELECT id, week_start, status, version, locked_at, approved_at, approved_by, created_at

    **scheduler.go** (new file, per Pattern 3 in research):

    Create `StartScheduler(ctx context.Context, pool *pgxpool.Pool)` following the receipt.StartWorker goroutine pattern:
    - Log "cutoff scheduler: starting (15m tick)"
    - Run `runCutoffCheck(ctx, pool)` immediately
    - Start `time.NewTicker(15 * time.Minute)`
    - Select on ctx.Done() or ticker.C
    - On ctx.Done(): log "cutoff scheduler: shutting down"

    `runCutoffCheck(ctx, pool)`:
    - Call GetCutoffConfig. If nil (no config), return silently.
    - Load timezone: `loc, err := time.LoadLocation(config.Timezone)` -- if error, log and return
    - Parse cutoff_time: split by ":" to get hour/minute
    - Compute next cutoff occurrence:
      - Get current time in loc: `now := time.Now().In(loc)`
      - Find most recent occurrence of day_of_week + cutoff_time in loc timezone
      - If now is past that occurrence AND it's within the current week window: cutoff has passed
    - Find current draft PO: `SELECT id FROM purchase_orders WHERE status = 'draft' ORDER BY week_start DESC LIMIT 1`
    - If draft exists and cutoff has passed: call LockPO(ctx, pool, poID)
    - Log result

    IMPORTANT: Use time.LoadLocation for DST-safe computation (per Pitfall 1). Never use fixed UTC offsets.

    **handler.go** -- Add these handler functions:

    `GetCutoffConfigHandler(pool)` -- GET, returns config or empty object
    `UpsertCutoffConfigHandler(pool)` -- PUT, admin-only (per D-01), body: {day_of_week, cutoff_time, timezone}
    `SimulateCutoffHandler(pool)` -- POST, admin-only (per D-04/D-05/D-06), calls LockPO on current draft
    `LockPOHandler(pool)` -- POST /orders/{id}/lock, admin-only
    `UnlockPOHandler(pool)` -- POST /orders/{id}/unlock, admin-only
    `ApprovePOHandler(pool)` -- POST /orders/{id}/approve, admin-only (per D-10)
    `GetOrdersByStatusHandler(pool)` -- GET /orders?status=locked, returns PO by status filter

    Map service errors to HTTP: ErrPONotDraft->409, ErrPONotLocked->409, ErrUnlockAfterApproval->409, ErrActiveShoppingListExists->409.

    `GetOrdersByStatusHandler`: read `status` query param, call GetOrdersByStatus, return the PO with its line items (call GetOrderLineItems for the matched PO). No admin restriction -- all authenticated users can view locked POs.
