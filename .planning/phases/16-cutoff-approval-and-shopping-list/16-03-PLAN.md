    Add these functions to service.go:

    **GetActiveShoppingList(ctx, pool)** -- Returns the active shopping list with vendor sections and items, or nil if none.
    ```sql
    SELECT sl.id, sl.po_id, po.week_start, sl.status, sl.assigned_to, sl.assigned_role, sl.created_at, sl.completed_at
    FROM shopping_lists sl
    JOIN purchase_orders po ON po.id = sl.po_id
    WHERE sl.status = 'active'
    LIMIT 1
    ```
    Then load vendor sections:
    ```sql
    SELECT svs.id, svs.shopping_list_id, svs.vendor_id, svs.vendor_name, svs.status, svs.completed_by, u.display_name, svs.completed_at
    FROM shopping_list_vendor_sections svs
    LEFT JOIN users u ON u.id = svs.completed_by
    WHERE svs.shopping_list_id = $1
    ORDER BY svs.vendor_name
    ```
    Then load items per section:
    ```sql
    SELECT sli.id, sli.shopping_list_id, sli.vendor_section_id, sli.purchase_item_id, sli.item_name, sli.photo_url, sli.store_location, sli.quantity, sli.unit, sli.checked, sli.checked_by, u.display_name, sli.checked_at
    FROM shopping_list_items sli
    LEFT JOIN users u ON u.id = sli.checked_by
    WHERE sli.shopping_list_id = $1
    ORDER BY sli.item_name
    ```
    Assemble into ShoppingList struct with nested VendorSections and Items.

    **GetShoppingListByID(ctx, pool, listID)** -- Same as above but filters by list ID instead of status. Used for history view.

    **GetShoppingListHistory(ctx, pool)** -- Returns past shopping lists (status='completed') with vendor sections (no items -- items loaded on expand):
    ```sql
    SELECT sl.id, sl.po_id, po.week_start, sl.status, sl.created_at, sl.completed_at
    FROM shopping_lists sl
    JOIN purchase_orders po ON po.id = sl.po_id
    WHERE sl.status = 'completed'
    ORDER BY po.week_start DESC
    ```
    For each list, load vendor sections with completion info (same query as above).
    Also compute missing_count per vendor section: `SELECT COUNT(*) FROM shopping_list_items WHERE vendor_section_id = $1 AND checked = false`.

    **CheckShoppingItem(ctx, pool, itemID string, checked bool, userID string)** -- Toggle check state:
    ```sql
    UPDATE shopping_list_items
    SET checked = $2, checked_by = CASE WHEN $2 THEN $3::uuid ELSE NULL END, checked_at = CASE WHEN $2 THEN now() ELSE NULL END
    WHERE id = $1
    RETURNING id
    ```
    Return error if 0 rows.

    **UpdateShoppingItemLocation(ctx, pool, itemID string, storeLocation string)** -- Update store_location on both shopping_list_items AND purchase_items (so future lists see it too):
    ```sql
    UPDATE shopping_list_items SET store_location = $2 WHERE id = $1 RETURNING purchase_item_id
    ```
    Then: `UPDATE purchase_items SET store_location = $2 WHERE id = $purchaseItemID`

    **UpdateShoppingItemPhoto(ctx, pool, itemID string, photoURL string)** -- Update photo_url on both shopping_list_items AND purchase_items (per Pitfall 6, SHOP-08):
    ```sql
    UPDATE shopping_list_items SET photo_url = $2 WHERE id = $1 RETURNING purchase_item_id
    ```
    Then: `UPDATE purchase_items SET photo_url = $2 WHERE id = $purchaseItemID`

    **CompleteVendorSection(ctx, pool, sectionID string, userID string)** (per D-21 through D-24):
    - BEGIN TX
    - `UPDATE shopping_list_vendor_sections SET status = 'completed', completed_by = $2, completed_at = now() WHERE id = $1 AND status = 'pending' RETURNING shopping_list_id`
    - If 0 rows: return error (already completed)
    - Check if all sections for this list are completed:
      ```sql
      SELECT COUNT(*) FROM shopping_list_vendor_sections WHERE shopping_list_id = $listID AND status = 'pending'
      ```
    - If count = 0 (all done): update shopping list and PO to completed:
      ```sql
      UPDATE shopping_lists SET status = 'completed', completed_at = now() WHERE id = $listID
      UPDATE purchase_orders SET status = 'completed' WHERE id = (SELECT po_id FROM shopping_lists WHERE id = $listID)
      ```
    - After completing a vendor section, call `NotifyVendorComplete(ctx, pool, listID)` (per SHOP-06)
    - COMMIT
    - Return whether list is now fully completed (bool)

    **NotifyVendorComplete(ctx, pool, listID string)** -- Phase 17 alert stub (per SHOP-06, research anti-pattern guidance):
    ```go
    // NotifyVendorComplete is a no-op stub for Phase 17 alert wiring.
    // Phase 17 will replace this with actual Zoho Cliq / email delivery.
    func NotifyVendorComplete(ctx context.Context, pool *pgxpool.Pool, listID string) error {
        log.Printf("alert pending (Phase 17): vendor section completed for shopping list %s", listID)
        return nil
    }
    ```
    This must be called from CompleteVendorSection after the section is marked completed but before COMMIT, so Phase 17 can wire in real alerts without restructuring the transaction flow.
