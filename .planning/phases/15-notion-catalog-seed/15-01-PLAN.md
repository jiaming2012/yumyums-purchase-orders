Create `backend/cmd/import-notion/main.go` (per D-03) -- a standalone Go command:

```
go run ./cmd/import-notion/ \
  --csv "/path/to/All Items ...csv" \
  --images "/path/to/All Items/" \
  --output "internal/inventory/fixtures/purchase_items.yaml"
```

**Flags:**
- `--csv` (required): path to Notion CSV file
- `--images` (required): path to `All Items/` directory containing image subdirectories
- `--output` (required): output YAML path (default: `internal/inventory/fixtures/purchase_items.yaml`)
- `--dry-run`: skip DO Spaces upload, use placeholder URLs

**Environment variables for DO Spaces** (same as server uses):
- `DO_SPACES_KEY`, `DO_SPACES_SECRET`, `DO_SPACES_ENDPOINT`, `DO_SPACES_REGION`, `DO_SPACES_BUCKET`

**Logic:**
1. Parse CSV using `encoding/csv`. Skip header row.
2. For each row:
   a. `Name` column -> `description` (per D-07). Apply title case normalization matching `normalizeItemName()` -- use `cases.Title(language.English)`.
   b. `Full Name` column -> `full_name` (per D-08). If Name is empty, use Full Name for description too (per D-09).
   c. `Category` column -> map to group name using D-01 mapping. If category is empty or unrecognized, assign to "Dry Goods" (per specifics).
   d. `Store` column -> `store_location`. Title-case normalize.
   e. `Photo` column -> URL-decode to get relative path. Find the image file in `--images` directory. The Photo value is like `All%20Items/Brisket%20Rolls/Screenshot_...png` -- URL-decode it, strip the `All Items/` prefix, combine with `--images` flag path to get the absolute file path.
3. For each item with a photo file that exists on disk:
   a. Reuse `photos.SpacesConfig` struct for configuration (per D-06), but use a direct `s3.Client` with `PutObject` instead of the presigned PUT pattern. D-06 references the "same presigned PUT pattern from Phase 12" but presigned URLs are designed for browser-side uploads; this server-side script should use direct `PutObject` which is simpler and more appropriate for a batch CLI tool.
   b. Upload key: `items/{slug}.png` where slug is kebab-case of the item description (lowercase, spaces to hyphens, strip special chars)
   c. Call `client.PutObject` with the file bytes, ContentType `image/png`, ACL `public-read`
   d. Record public URL: `photos.PublicURL(endpoint, bucket, key)`
   e. If photo file doesn't exist or Photo column is empty, set `photo_url` to empty string (omit from YAML)
4. Group items by their mapped group name.
5. Write YAML to `--output` in this structure (matching existing fixture patterns):

```yaml
# Generated from Notion CSV export. DO NOT EDIT.
# Re-run: go run ./cmd/import-notion/ --csv ... --images ... --output ...
items:
  - group: "Proteins"
    purchase_items:
      - description: "Brisket"
        full_name: "USDA Choice Brisket - whole"
        photo_url: "https://nyc3.digitaloceanspaces.com/bucket/items/brisket.png"
        store_location: "Restaurant Depot"
```

6. Print summary: total items, items with photos uploaded, items without photos, items per group.

**Important:** The conversion script uses direct `PutObject` (not presigned URLs per D-06 deviation noted above). Import `s3` directly and create the S3 client inline in the script:
```go
client := s3.New(s3.Options{
    Region:       region,
    Credentials:  credentials.NewStaticCredentialsProvider(key, secret, ""),
    BaseEndpoint: aws.String(endpoint),
    UsePathStyle: true,
})
```
Then call `client.PutObject(ctx, &s3.PutObjectInput{...})`.

After the script is written and compiles, the user will run it manually with the correct env vars to generate the YAML. The generated YAML IS committed (per D-05).

For now, create the script and also create a placeholder `purchase_items.yaml` with a comment header and an empty `items: []` so the codebase compiles. The real YAML will be generated when the user runs the script.
