## Decisions Made
- `nilIfEmpty` converts empty strings to nil pointers so optional columns (photo_url, store_location, full_name) remain NULL in Postgres rather than empty strings — matches how these columns are used by existing inventory handlers
- Group lookup by name (SELECT not JOIN) is the same pattern used in the import-notion script; consistent and simple
- Warning-and-skip for unknown groups prevents a missing Notion category from breaking all other seeds
