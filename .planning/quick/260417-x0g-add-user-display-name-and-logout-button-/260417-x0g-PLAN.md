Create tests/index.spec.js with these tests:

1. Use the same login helper pattern as onboarding.spec.js:
   ```js
   const ADMIN_EMAIL = 'jamal@yumyums.kitchen';
   const ADMIN_PASSWORD = 'test123';
   async function login(page, email, password) {
     await page.goto('/login.html');
     await page.fill('input[type="email"]', email || ADMIN_EMAIL);
     await page.fill('input[type="password"]', password || ADMIN_PASSWORD);
     await page.click('button.btn');
     await page.waitForURL(url => !url.pathname.includes('login'));
   }
   ```

2. Test: "index.html shows user display name after login"
   - login(page)
   - Navigate to /index.html
   - Wait for `.user-bar` to appear (the /api/v1/me fetch needs to complete)

3. Test: "logout button redirects to login.html"
   - login(page)
   - Navigate to /index.html
   - Wait for `.user-bar` to appear

   - Assert current URL includes login.html

   - waitForURL to contain 'login.html' (the checkAuth 401 redirect fires because session cookie was cleared by logout POST)
   - This is the key "state-flush" test: proves the server-side session is actually destroyed
